# 问题回顾

* 无法打开图片： 原因是用字符串拼接路径导致路径格式错误，cv2.imread 返回 None，后续 cvtColor 断言失败。
* 程序崩溃（OverflowError）： Hough 返回的坐标被转换为 uint16，在做加减运算时出现下溢/越界（负索引被解释为大正数）。
* 保存失败风险： 结果目录未确保存在，imwrite 可能失败或写到错误位置。
* 硬编码路径与分辨率依赖： 使用绝对路径与高分辨率图片会导致性能或读取问题。
* 相关修改文件： main.py

# 已采取的修复（原理）

* 用 os.path.join 构建路径： 保证跨平台的正确分隔符，避免拼接错误。
* 检查 img is None： 在对图像做任何处理前确认加载成功，防止 OpenCV 断言失败。
* 将坐标转换为 Python int 并加边界检查： 避免 uint16 下溢、负索引和越界访问导致的异常。
* 确保 result 目录存在： 使用 os.makedirs(..., exist_ok=True) 再保存结果。

# 当前方法（本项目）：
* 颜色分割（HSV）： 将 BGR 转到 HSV，按颜色阈值生成红/黄/绿掩码。
* 圆检测（HoughCircles）： 在各颜色掩码上运行霍夫圆变换定位灯泡候选圆。
* 掩码验证： 在圆心邻域采样掩码像素密度（h/s 比值）以确认该圆是否为对应颜色灯。
* 标注输出： 在原图绘制圆和文本并保存结果。
# 可行的改进方向：
* 传统图像处理提升： 预处理（高斯模糊、形态学闭运算）、自适应阈值、按 ROI 限制* 搜索、用轮廓/连通组件替代或结合霍夫检测。
基于机器学习/深度学习： 用轻量检测网络（例如 YOLO、SSD 或 Faster R-CNN）训练交通灯检测与分类，鲁棒性更强，适应不同拍摄角度与光照。
时序与跟踪： 在视频中使用跟踪（Kalman/ SORT）结合检测结果提高稳定性并减少误报。
* 颜色归一化与光照校正： 使用色彩恒常性方法或对比度限制自适应直方图均衡（CLAHE）改善弱光下的颜色分割。
* 标注与数据： 收集/标注多样化样本以调优阈值或训练模型，支持不同灯具形状（圆、方、竖排灯组）。
